# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make run        # Run the app (port 8080)
make dev        # Run with hot reload via air
make swag       # Regenerate Swagger docs from annotations
go mod tidy     # Tidy dependencies
```

Swagger UI is served at `http://localhost:8080/swagger/index.html`.

After modifying any `// @...` annotation in handlers or `main.go`, run `make swag` to regenerate `docs/`.

## Architecture

The project uses a 3-layer architecture with a swappable API transport layer:

```text
cmd/app/main.go                        → wires dependencies, selects transport
internal/api/server.go                 → Server interface (the swap point)
internal/api/http/                     → Gin HTTP implementation
  server.go                            → route registration & server lifecycle
  todo_handler.go                      → request handlers with Swagger annotations
  todo_requests.go                     → request body structs (CreateTodoRequest, UpdateTodoRequest)
  error_handler.go                     → AppError → HTTP status mapping, validation error formatting
internal/services/                     → business logic
internal/repositories/
  todo_repo.go                         → TodoRepository interface
  todo_local_repo.go                   → in-memory implementation (uses UUID for IDs)
internal/models/
  todo.go                              → Todo struct
  errors.go                            → sentinel errors (e.g. ErrTodoNotFound)
internal/errs/
  app_error.go                         → AppError type, error codes, constructors (NotFound, Internal)
docs/                                  → auto-generated by swag, do not edit manually
```

### Error handling

Errors flow through two layers:

- **Repository/Service**: return sentinel errors from `internal/models/errors.go` (e.g. `models.ErrTodoNotFound`)
- **Service**: wraps them into `*errs.AppError` (via `errs.NotFound`, `errs.Internal`) with machine-readable codes and safe user messages
- **HTTP handler**: calls `handleAppError(c, err)` in `error_handler.go` which maps `AppError.Code` → HTTP status

### API abstraction

`internal/api/server.go` defines a single `Server` interface. The only concrete implementation is `internal/api/http/` (Gin). To add a new transport (e.g. GraphQL), create `internal/api/graphql/` implementing `Server` and change one line in `main.go`. Services and repositories are never touched when swapping transports.

### Adding a new endpoint

1. Add handler method to `internal/api/http/todo_handler.go` with `// @...` Swagger annotations
2. Add any new request structs to `internal/api/http/todo_requests.go`
3. Register the route in `internal/api/http/server.go` → `registerRoutes()`
4. Run `make swag`

### go tool vs go install

`swag` and `air` are declared as `tool` dependencies in `go.mod` and invoked via `go tool <name>`, not installed globally.
